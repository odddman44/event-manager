# 이벤트 매니저 MVP PRD

## 핵심 정보

**목적**: 주최자가 이벤트를 만들고 링크를 공유하면 참여자가 회원가입 없이 즉시 참여 의사를 밝힐 수 있도록 해 카카오톡 기반의 비효율적 참여자 관리를 해결한다
**사용자**: 수영, 헬스, 친구 모임 등 소규모 이벤트를 반복적으로 주최하는 개인 주최자 및 그 참여자

---

## 사용자 여정

### 주최자 흐름

```
1. 랜딩 페이지
   ↓ [시작하기 / 로그인] 클릭

2. 로그인 페이지 (미가입 시 → 회원가입 페이지)
   ↓ 로그인 성공

3. 내 이벤트 목록 페이지 (주최자 홈)
   ↓ [새 이벤트 만들기] 클릭

4. 이벤트 생성 페이지
   ↓ 제목/날짜/장소/정원 입력 후 [생성] 클릭

5. 이벤트 관리 페이지
   → 공유 링크 복사 → 카카오톡 전송
   → 참여자 목록 실시간 확인
   → [내 이벤트 목록으로] 클릭 → 3번으로 복귀
```

### 참여자 흐름

```
1. 카카오톡 등에서 공유 링크 클릭 → 참여 페이지
   ↓ 이름 + 메모(선택) 입력 후 [참여하기] 클릭

2. 참여 완료 화면 (guest_token → localStorage 저장)
   ↓ 재방문 시 자동 인식

   [기존 참여자] → 참여 페이지 (메모 수정 / 참여 취소 가능)
   [새 참여자]   → 참여 페이지 (이름 + 메모 입력)
```

### 어드민 흐름

```
1. 로그인 페이지 (admin 계정)
   ↓ 로그인 성공 → admin role 확인

2. 어드민 대시보드
   → 통계 카드: 총 가입자 / 총 이벤트 / 총 참여자 수 확인

3. 어드민 이벤트 목록 페이지 (이벤트 삭제 가능)
   또는
   어드민 사용자 목록 페이지 (사용자 삭제 가능)
```

## 기능 명세

### 1. MVP 핵심 기능

| ID       | 기능명              | 설명                                                                                                | MVP 필수 이유                                                   | 관련 페이지               |
| -------- | ------------------- | --------------------------------------------------------------------------------------------------- | --------------------------------------------------------------- | ------------------------- |
| **F001** | 이벤트 생성         | 제목, 날짜, 장소, 정원을 입력해 이벤트를 등록하고 share_token을 자동 발급                           | 서비스의 출발점이자 주최자의 핵심 행위                          | 이벤트 생성 페이지        |
| **F002** | 내 이벤트 목록 조회 | 주최자가 자신이 만든 이벤트 목록을 카드 형태로 확인                                                 | 주최자가 이벤트를 관리하기 위한 진입점                          | 내 이벤트 목록 페이지     |
| **F003** | 이벤트 관리         | 참여자 목록 조회 + 공유 링크 복사 기능을 하나의 페이지에서 제공                                     | 주최자가 참여 현황을 파악하고 링크를 배포하는 핵심 기능         | 이벤트 관리 페이지        |
| **F004** | 비회원 참여 신청    | 공유 링크(share_token)로 접근해 이름과 메모를 입력하면 참여 등록, guest_token을 localStorage에 저장 | 참여자가 회원가입 없이 즉시 참여할 수 있어야 서비스 가치가 성립 | 참여 페이지               |
| **F005** | 참여 상태 변경      | guest_token 기반으로 재방문 시 자동 인식하여 메모 수정 또는 참여 취소 처리                          | 참여 의사 변경이 가능해야 실제 이벤트 관리에 유용               | 참여 페이지               |
| **F006** | 플랫폼 통계 조회    | 총 가입자 수, 총 이벤트 수, 총 참여자 수를 카드 형태로 표시                                         | 플랫폼 운영 현황 파악을 위한 어드민 핵심 기능                   | 어드민 대시보드           |
| **F007** | 전체 이벤트 관리    | 플랫폼 전체 이벤트 목록 조회 및 삭제                                                                | 부적절한 이벤트 제거를 위한 운영 필수 기능                      | 어드민 이벤트 목록 페이지 |
| **F008** | 전체 사용자 관리    | 플랫폼 전체 사용자 목록 조회 및 삭제                                                                | 계정 관리를 위한 운영 필수 기능                                 | 어드민 사용자 목록 페이지 |

### 2. MVP 필수 지원 기능

- 프로필 상세 관리 (아바타, 자기소개)

## 페이지별 상세 기능

### 랜딩 페이지

> **구현 기능:** `F010` | **접근:** 전체 공개

| 항목            | 내용                                                                                                            |
| --------------- | --------------------------------------------------------------------------------------------------------------- |
| **역할**        | 서비스 소개 및 로그인/회원가입 진입점                                                                           |
| **진입 경로**   | 도메인 루트 접속, 로그아웃 후 자동 리다이렉트                                                                   |
| **사용자 행동** | 서비스 소개 확인 후 로그인 또는 회원가입 버튼 클릭                                                              |
| **주요 기능**   | - 서비스 핵심 가치 한 줄 소개<br>- **[로그인]** 버튼 → 로그인 페이지<br>- **[시작하기]** 버튼 → 회원가입 페이지 |
| **다음 이동**   | 로그인 → 로그인 페이지, 회원가입 → 회원가입 페이지, 이미 로그인 상태 → 내 이벤트 목록 페이지 자동 리다이렉트    |

---

### 로그인 페이지

> **구현 기능:** `F010` | **접근:** 비로그인 전용

| 항목            | 내용                                                                                                           |
| --------------- | -------------------------------------------------------------------------------------------------------------- |
| **역할**        | Supabase Auth를 통한 이메일/비밀번호 로그인 처리                                                               |
| **진입 경로**   | 랜딩 페이지 [로그인] 버튼, 보호된 경로 접근 시 자동 리다이렉트                                                 |
| **사용자 행동** | 이메일과 비밀번호 입력 후 로그인 버튼 클릭                                                                     |
| **주요 기능**   | - 이메일 / 비밀번호 입력 폼<br>- **[로그인]** 버튼<br>- 회원가입 페이지 링크<br>- 로그인 실패 에러 메시지 표시 |
| **다음 이동**   | 성공 (admin) → 어드민 대시보드, 성공 (user) → 내 이벤트 목록 페이지, 실패 → 에러 메시지 표시                   |

| **다음 이동**   | 새 이벤트 → 이벤트 생성 페이지, 이벤트 클릭 → 이벤트 관리 페이지                                                                                                                                                                   |

### 이벤트 생성 페이지

> **구현 기능:** `F001` | **인증:** 로그인 필수 (role: user)

| 항목            | 내용                                                                                                                                                                                        |
| --------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **역할**        | 이벤트 정보를 입력해 새 이벤트를 등록하고 share_token을 자동 발급                                                                                                                           |
| **진입 경로**   | 내 이벤트 목록 페이지 [새 이벤트 만들기] 버튼 클릭                                                                                                                                          |

| **주요 기능**   | - 제목 입력 필드 (필수)<br>- 날짜/시간 입력 필드 (필수)<br>- 장소 입력 필드 (선택)<br>- 최대 정원 입력 필드 (선택)<br>- **[이벤트 생성]** 버튼<br>- **[취소]** 버튼 → 내 이벤트 목록 페이지 |
| **다음 이동**   | 성공 → 이벤트 관리 페이지 (생성된 이벤트), 실패 → 에러 메시지 표시                                                                                                                          |

| 항목            | 내용                                                                                                                                                                                                                                   |
| --------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **역할**        | 이벤트의 참여자 현황 확인 및 공유 링크 배포 허브                                                                                                                                                                                       |
| **진입 경로**   | 이벤트 생성 완료 후 자동 이동, 내 이벤트 목록 페이지 이벤트 카드 클릭                                                                                                                                                                  |
| **사용자 행동** | 공유 링크를 복사해 참여자에게 전송하고, 참여자 목록을 확인                                                                                                                                                                             |
| **주요 기능**   | - 이벤트 정보 요약 표시 (제목, 날짜, 장소, 정원)<br>- 공유 링크 표시 + **[링크 복사]** 버튼 (클립보드 복사)<br>- 참여자 목록 테이블 (이름, 메모, 참여 상태, 신청 시각)<br>- 참여 인원 / 정원 카운터<br>- **[내 이벤트 목록으로]** 링크 |
| **다음 이동**   | 목록으로 → 내 이벤트 목록 페이지                                                                                                                                                                                                       |

### 참여 페이지

| --------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |

| **다음 이동**   | 참여 완료 → 완료 메시지 표시 (동일 페이지), 참여 취소 → 취소 완료 메시지 표시                                                                                                                                                                                                                                                    |

## 기술 스택

### 프론트엔드 프레임워크

- **Next.js 15** (App Router) — React 풀스택 프레임워크, Server Actions 활용
- **TypeScript 5.6+** — 타입 안전성 보장
- **React 19** — UI 라이브러리

### 스타일링 & UI

- **TailwindCSS v4** — 유틸리티 CSS 프레임워크 (설정 파일 없는 새로운 엔진)
- **shadcn/ui** — 고품질 React 컴포넌트 라이브러리
- **Lucide React** — 아이콘 라이브러리
- **디자인 방향**: 럭키밀 스타일 — 따뜻한 오렌지/코럴 계열 포인트 (`#FF6B35`), border-radius 16-24px, 넉넉한 여백

### 폼 & 검증

- **React Hook Form 7.x** — 폼 상태 관리
- **Zod** — 스키마 검증

### 백엔드 & 데이터베이스

- **Supabase** — 인증 (Auth), 데이터베이스 (PostgreSQL), RLS (Row Level Security)
- **RLS 핵심 규칙**:
  - events: 주최자 본인만 CRUD, share_token으로 SELECT 공개
  - participants: guest_token 소유자가 본인 행 UPDATE, 이벤트 주최자가 전체 SELECT

### 아키텍처 레이어

```
Server Action / Route Handler  (Controller)
  └── Service  (비즈니스 로직)
        └── Repository  (Supabase 쿼리)
```
