# 이벤트 매니저 MVP 개발 로드맵

주최자가 이벤트를 만들고 링크를 공유하면 참여자가 회원가입 없이 즉시 참여 의사를 밝힐 수 있는 서비스

## 현재 상태

- **진행 단계**: MVP 배포 완료 — 21/22 Tasks 완료 (Task 008-1 커버 이미지 업로드만 백로그로 남음)
- **최종 업데이트**: 2026-07-10

## 개요

이벤트 매니저는 소규모 이벤트를 반복적으로 주최하는 개인 주최자와 그 참여자를 위한 서비스로, 카카오톡 기반의 비효율적 참여자 관리를 대체합니다. 핵심 가치는 다음과 같습니다:

- **간편한 이벤트 생성**: 제목/날짜/장소/정원만 입력하면 공유 링크(share_token) 자동 발급
- **회원가입 없는 참여**: 참여자는 링크 클릭 후 이름/메모만으로 즉시 참여, guest_token으로 재방문 인식
- **실시간 참여 현황 관리**: 주최자는 한 페이지에서 참여자 목록 확인 및 링크 배포

### 개발 전략: UI-first

전체 UI 골격을 먼저 완성한 뒤 기능을 붙이는 방식을 채택합니다. 기존 Feature-by-feature 방식(기능 단위로 UI + 백엔드를 동시 구현)과의 차이점은 다음과 같습니다:

- **빠른 플로우 검증**: 더미 데이터로 전체 사용자 흐름을 조기에 체험하고 문제를 발견
- **역할 분리**: UI 작업과 백엔드 로직 구현이 명확히 분리되어 집중도 향상
- **중복 작업 최소화**: 공통 컴포넌트를 한 번만 개발하고 모든 페이지에서 재사용
- **변경 유연성**: 백엔드 연동 전 UI를 자유롭게 수정 — 비용 없는 피드백 반영

## 기술 스택

- **프론트엔드**: Next.js 15 (App Router), TypeScript 5.6+, React 19
- **스타일링/UI**: TailwindCSS v4, shadcn/ui, Lucide React (럭키밀 스타일, 포인트 컬러 `#FF6B35`, border-radius 16-24px)
- **폼/검증**: React Hook Form 7.x, Zod
- **백엔드/DB**: Supabase (Auth, PostgreSQL, RLS) — `@supabase/ssr`
- **아키텍처**: Controller(Server Action) → Service → Repository (레이어드)
- **배포**: Vercel

## 개발 워크플로우

1. **작업 계획**: 기존 코드베이스 상태를 파악하고 `ROADMAP.md`를 갱신하여 우선순위 작업을 결정
2. **작업 구현**: 각 Task의 체크리스트를 따라 구현하고, 단계 완료 시마다 진행 상황 갱신
3. **테스트 검증**: API 연동 및 비즈니스 로직 구현 시 Playwright MCP로 E2E 테스트 수행 후 다음 단계 진행
4. **로드맵 업데이트**: 완료된 Task와 세부 항목을 ✅로 표시

## 상태 표시 규칙
